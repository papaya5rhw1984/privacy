기기 안에서만 동작하고 외부로 어떤 데이터도 보내지 않는 앱.
✔ 이런 앱이면 여기: 인터넷 통신 없음 · 광고/분석 없음 · 로그인 없음 · 위치/카메라/연락처 권한 없음
예) 세계시간, 음악 플레이어, 오목, 기념일 D-day(알림은 기기 내부 처리)
📄 index.html 열기 (공통 문서)
제출 URL 예시: https://<배포주소>/index.html
🔵 유형② 인터넷 사용 · 무수집

외부 데이터를 받아오지만, 개인정보는 수집 안 하는 앱

INTERNET 권한으로 외부 API를 호출해 정보를 보여주지만, 이용자 개인정보는 수집·저장하지 않는 앱.
✔ 이런 앱이면 여기: INTERNET 권한 사용 · 외부 API/서버 조회 · 광고/분석 없음 · 로그인 없음
예) 환율 라이브, (앞으로) 버스·지하철 도착, 날씨
📄 type2-internet.html (범용 템플릿) 📄 livefx.html (환율 라이브 전용)
제출 URL 예시: https://<배포주소>/livefx.html (앱 전용) 또는 /type2-internet.html (범용)
🟠 유형③ 광고 · 분석 · 결제

광고·분석 도구·인앱결제가 들어간 앱

AdMob 광고, 분석(Analytics), 또는 인앱결제를 사용해 광고 식별자 등 일부 정보가 처리되는 앱. ※도입 시 스토어의 데이터 안전·연령등급도 함께 갱신해야 함.
✔ 이런 앱이면 여기: AdMob 등 광고 SDK · Google Analytics 등 분석 · 인앱결제/구독
(현재 우리 앱은 해당 없음 — 나중에 광고/결제 붙일 때 사용)
📄 type3-ads.html (범용 템플릿)
제출 URL 예시: https://<배포주소>/type3-ads.html · 사용 시 [앱 이름]·실제 사용하는 SDK명으로 채우기

🚀 쓰는 법

  1. 위 1초 선택표로 내 앱의 유형을 고른다.
  2. 해당 문서를 연다. 범용 템플릿(type2/type3)이면 [앱 이름] 부분을 실제 앱 이름으로 바꿔 앱이름.html로 복사 저장.
  3. 이 privacy 폴더를 배포(깃 푸시/Netlify)하면 https://<배포주소>/문서이름.html URL이 생긴다.
  4. 스토어 등록 화면의 개인정보처리방침 URL 칸에 그 주소를 붙여넣는다.
